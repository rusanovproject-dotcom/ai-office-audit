# AI Office Audit

Сайдкар-аудит для твоего **уже настроенного AI-офиса на Claude Code**.

Приходит сбоку, за одну живую сессию снимает картину твоей реальной работы, находит дыры
на твоих же файлах — и **ничего не ломает**: аудит read-only, любая правка только с твоего «да»
и после слепка, к которому всегда можно откатиться.

Что проверяет:

- **`/audit-security`** — секреты в транскриптах, git-истории и правах. Транскрипты Claude Code
  (`~/.claude/projects/*.jsonl`) не смотрит ни один инструмент, а туда оседают вставленные в чат
  ключи и тела прочитанных агентом `.env`. Показывает **категориями и счётчиками — без тел ключей**.
- **`/audit-fit`** — «что ты делаешь руками, хотя под это уже есть твой же скилл». Единица метрики —
  сессия, а не tool-call. Машинные прогоны (SDK/боты/cron) отсечены.
- **`/audit-memory`** — контур памяти офиса: живая ли память, поспевает ли дистилляция, не молчит ли
  enforcement.
- **`/audit-dev`** — как у тебя устроен цикл разработки, и встройка готового органа (Кузница/forge).

## Установка

```bash
git clone https://github.com/rusanovproject-dotcom/ai-office-audit.git
cd ai-office-audit
claude
```

Открылась сессия Claude Code **внутри этой папки** — скиллы и защита подхватятся сами.
Дальше одной фразой:

```
/audit-security --target ~/путь-к-твоему-офису
```

(или просто «проверь безопасность моего офиса ~/path» — скилл поймёт).

Так же запускаются `/audit-fit`, `/audit-memory`, `/audit-dev`.

> Запускай `/audit-*` **из папки клона** и указывай `--target` **абсолютным** путём. Корень пака
> опознаётся по файлу-маркеру `.ai-office-audit` (не удаляй и не переименовывай). Из каталога
> проверяемого офиса скилл не запускается — движок берётся только из доверенного клона.

## Почему это безопасно

Аудит идёт в **этой** папке, твой офис подключается только на чтение по `--target`. Мы в него не
ставимся (наши скиллы не столкнутся с твоими).

1. **Read-only на уровне прав.** PreToolUse-хук (`hooks/readonly-gate.sh`, зарегистрирован в
   `.claude/settings.json`) блокирует любую запись в твой офис по умолчанию.
2. **Слепок до первого касания.** Прежде чем что-то менять, снимается контентный слепок
   (`cp -a`, а не git-ветка) — он ловит и незакоммиченное. Откат возвращает офис целиком.
3. **Одна правка за раз, с твоего «да».** Опции: `да / только важное / по номерам / нет /
   «это осознанно, не баг»`. «Осознанно» → находка уходит в раздел «выглядит криво, но работает»,
   а не в фикс.
4. **Ничего не уносится наружу.** В любой сохранённый отчёт идут только категории, счётчики и
   `файл:строка` — никаких тел секретов и никакого твоего текста.

Твой стиль (структура папок, форма памяти, имена) **не режет оценку** — это флаг «house style»,
опция, а не дефект. Балл роняют только универсальные дыры: утечка, мёртвый роутинг, орфаны, дубли.

## Требования

- [Claude Code](https://claude.com/claude-code)
- `jq`, `git`; стандартные `grep`/`find`/`rsync`. `bats` — только чтобы прогнать тесты. macOS и Linux.

## Проверить, что всё живо

```bash
bats tests/
```
